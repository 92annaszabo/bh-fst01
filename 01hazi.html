Elméleti kérdések

A
-Mi az az URL?
Az URL egy webcím. Ez alapján tudjuk beazonosítani milyen oldalt szeretnénk megnyitni, a böngészőnk címsorában.
Ez is egy mozaikszó: Uniform Resource Locator. Vagyis egységes erőforrás hely.
Több részből épül fel: protokoll,tartománynév,portszám,elérési út.
-Mi az az IP cím?
Az Internet Protocol egy egyedi hálózati azonosító. Minden végpontnak a hálózaton saját IP címe van.
Van IPv4(32bites) és IPv6(128bites) cím. Ez alapján találjuk meg a weboldalakat is például.
-Mi az a domain név?
A domain név a szerver beazonosítására létrehozott névtartomány. Az IP cím szöveges megfelelője.
Több szintű. Top-level domain, second-level domain, third-level domain. Őt azonosítja be a DNS szerver.
-Mi az az aldomain?
Ponttal elválasztott, összetettebb domain név első része. Példaként:
Ha van egy www.blog.homework.hu címünk akkor a blog. lesz az első fele vagyis az aldomain címünk.
-Mi az a Domain Name Service?
A Domain Name Service azonosítja a webcímünket és navigál el a kívánt szerverhez.
Mindez hierarchikus felépítésű mint akármilyen hálózati rendszer vagyis szerverről szerverre lépkedve jutunk el a kívánt helyre.
-Mi az az IANA és mi a feladata?
Internet Assigned Numbers Authory
Az ő feladatuk az internet zökkenőmentes működése. Ők koordinálnak néhány kulcsfontosságú elemet. Ilyenek például:
Domain Name Services
IP címekért és az autonóm rendszerszámokért a zökkenőmentes internetes forgalomirányításhoz.
Az internetes protokollok számozási rendszerért a szabványügi testületekkel együttműködve.
-Mi az az Internet Szolgáltatók Tanácsa és mi a feladata?
Az internet szolgáltatók tanácsa egy hálózati koordinációs központ.
Feladatuk az alábbiak:

különös tekintettel az irányadó nemzetközi standardokra.
3. Domain név regisztrációs feladatokat lát el.

A BIX idális forgalom kicserélő központ. Hozzá csatlakozik be az összes magyarországi internet szolgáltató.
Ezen keresztül csatlakozunk rá az internet gerinchálózatára Magyarországon.

-Mi az a HTML?
Hyper Text Markup Language vagyis hiperszöveges jelölőnyelv.
Leíró nyelv melynek a W3C internetes szabványoknak kell megfelelnie. Mindenki számára elérhető tartalmat lehet vele létrehozni.
-Írd le a HTML dokumentum általános szerkezetét!
Áll egy dokumentum típus definícióból a legelején, ahol megadjuk milyen HTML-ben írjuk.
Áll egy html nevű html elemből vagyis ezen belül teljes egészében html dokumentumról van szó.
Áll egy head-ből vagyis fejlécből, mely adatról szólnak
Áll egy body nevű html szövegtörzsből.
-Mi az a Document Type Definition?
A DTD-vel definiáljuk a struktúrát és az elemeket, vagyis adjuk meg miben írjuk a kódot.
-Mi az a head?
A head a fejléc, ami adatot tartalmaz az oldalról. Magán az oldalon információt nem látunk róla csak a böngésző kis fülecskéjén. Nálam ez a 01 Homework.
-Mi az a body?
A body a szövegtörzs. Ez tartalmazza az oldalon megjelenített tartalmat.
-Mik azok a metaadatok?
A meta adatok, tagok azon információkat leírok adatok az oldal adatairól. Ilyen lehet például az oldal szerzője, rövid leírása.
-Mi az a title és hova írhatjuk? Mi az a br?
A head elembe vagyis a fejlécbe írhatjuk, a title lesz az oldal rövid címe ami megjelenik nekünk a böngésző kis fülén.
A br elem a sortörés elem amit mindig használnunk kell ha sortörést szeretnénk, mert egy sima enter nem elegendő a kódszerkesztőben.
A br elemnek nincs záróeleme!
C
-Mik azok a HTML elemek?
A HTML elemek a dokumentum szerkezeti elemei, ezekből épül fel hierarcikus alapon a tartalom. Például: html,head,body,title,section,stb.
-Mik azok a HTML tagek?
A HTML tagek a html-t leíró formázó utasításai. Minden (majdnem minden) tagnek van záró utasítása.
-Mit jelent az, hogy egy tag páros vagy páratlan?
Például egy szabálynál megmondhatjuk, hogy minden páros paragrafus piros színű legyen, vagy egy táblázatba,felsorolásba minden páros sor zöld minden páratlan piros.
-Mit jelent az, hogy a HTML dokumentum fa szerkezetű?
Azt, hogy hierarcikus felépítésű. Először definiáljuk, utána megadjuk a html elemet, utána címsort, azzal egyenrangú a szövegtörzs, stb. így építkezve.
Vagyis az elemek egymásba ágyazva vagy egymás mellett jelennek meg.
-Mi a különbség a DOM és a forráskód között?
A Document Objectum Model egy szabványos interface, amelyen keresztül kapcsolatot lehet létesíteni JavaScriptel.
A forráskód pedig a leíró nyelv (jelen esetben ez a HTML) definícióinak és utasításain sorozatát tartalmazza, vagyis a fordítóprogram dokumentumként jeleníti meg nekünk vagy más esetben szoftverként.
A DOM-ot a böngésző használja, fordítja le a forráskódot pedig a fejlesztő írja.
-Mit jelent, hogy egy elem egy másiknak a gyermeke?
A gyökér html eleme a head és a body, ezeknek lehet például a gyereke a h1, vagyis a tagek egymásba ágyazását jelenti.
-Mi az a gyökérelem? Mi az a HTML validáció?
A gyökér elem a html elem. Ez jelent a legfelső szintű elemet, ez adja meg a html dokumentum gyökerét. Minden többi elem ennek az elemnek a gyermeke.
A HTML validáció azt jelenti, hogy egy HTML kód megfelel az adott HTML szabványoknak. Különböző HTML verziók nem biztos hogy validált a régebbi szabványok szerint.
-Mi az a szemantikus HTML, miért fontos?
Kifejezéssel bíró HTML elemek. Fontos azért lehet, hogy ha valaki látás sérült például, akkor amikor egy program felolvassa a szöveget,
különböző hangsúllyal lehetséges. Például ez egy szemantikus elem.
